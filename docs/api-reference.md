# API Reference — Recommendation Service

Base URL: `http://localhost:8085`

All endpoints require a user JWT:
```
Authorization: Bearer <access_token>
```

Tokens are issued by the user-microservice and validated here using the shared `jwt.secret`.

---

## Recommendations — `/api/recommendations`

### GET /api/recommendations

Fetch personalised movie recommendations for the authenticated user.

**Authenticated** · Returns `200 OK`

Recommendations are generated by combining collaborative filtering (movies rated ≥ 4 by similar users) and content-based filtering (movies sharing genres with the user's highly-rated movies). Results from both approaches are merged, deduplicated, and sorted by relevance score descending.

**Query parameters**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `genre` | String | — | Filter results to movies containing this genre (case-sensitive) |
| `yearFrom` | Integer | — | Include only movies with `releaseYear ≥ yearFrom` |
| `yearTo` | Integer | — | Include only movies with `releaseYear ≤ yearTo` |
| `page` | Integer | `0` | Zero-based page index |
| `size` | Integer | `20` | Page size — min 1, max 100 |

Filters are optional and can be combined freely. When `genre` is provided, only movies whose `genres` list contains that exact string are returned. `yearFrom` and `yearTo` can be used independently or together as a range.

**Validation rules**

| Parameter | Rules |
|-----------|-------|
| `page` | Must not be negative |
| `size` | Must be between 1 and 100 inclusive |

**Response**
```json
{
  "content": [
    {
      "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
      "title": "Inception",
      "genres": ["Science Fiction", "Action", "Thriller"],
      "releaseYear": 2010,
      "description": "A thief who steals corporate secrets through dream-sharing technology.",
      "posterUrl": "https://example.com/inception.jpg",
      "averageRating": 4.7,
      "relevanceScore": 5
    }
  ],
  "page": 0,
  "size": 20,
  "totalElements": 1,
  "totalPages": 1
}
```

**Response fields**

| Field | Type | Description |
|-------|------|-------------|
| `content` | Array | Page of recommendation objects |
| `page` | Integer | Current page index |
| `size` | Integer | Requested page size |
| `totalElements` | Long | Total number of recommendations after filters |
| `totalPages` | Integer | Total number of pages |

**RecommendationResponse fields**

| Field | Type | Description |
|-------|------|-------------|
| `id` | String (UUID) | Movie identifier |
| `title` | String | Movie title |
| `genres` | Array<String> | Genre list |
| `releaseYear` | Integer | Year of release |
| `description` | String | Movie description |
| `posterUrl` | String | Poster image URL |
| `averageRating` | Double | Current average rating across all users |
| `relevanceScore` | Long | Combined relevance from both algorithms — higher means more strongly recommended |

**Errors**

| Status | Reason |
|--------|--------|
| 400 | `page` is negative |
| 400 | `size` is less than 1 or greater than 100 |
| 401 | Missing or invalid JWT |
| 500 | Unexpected server error |

## Recommendation Algorithms

### Relevance scoring

A movie can appear in one or both result sets:

- **Collaborative only** — relevance = number of similar users who rated it ≥ 4
- **Content-based only** — relevance = number of genres overlapping with the user's liked genres
- **Both** — relevance = sum of the two scores

Movies in both sets rank higher than those found by a single approach.

### No ratings yet

If the authenticated user has no ratings with score ≥ 4, both algorithms return empty results. The response will have an empty `content` array and `totalElements: 0`.

## Data Model

### RecommendationResponse

| Field | Type | Notes |
|-------|------|-------|
| `id` | String (UUID) | Movie node `id` from Neo4j |
| `title` | String | Movie title |
| `genres` | List<String> | Stored as a list property on the `Movie` node |
| `releaseYear` | Integer | Used for `yearFrom`/`yearTo` filtering |
| `description` | String | May be null if not populated |
| `posterUrl` | String | May be null if not populated |
| `averageRating` | Double | May be null if no ratings exist yet |
| `relevanceScore` | Long | Computed at query time — not stored |

### PagedRecommendationResponse

| Field | Type | Notes |
|-------|------|-------|
| `content` | List<RecommendationResponse> | The recommendations for this page |
| `page` | Integer | Zero-based page index from the request |
| `size` | Integer | Page size from the request |
| `totalElements` | Long | Total matching items across all pages |
| `totalPages` | Integer | `ceil(totalElements / size)` |

```json
{
  "type": "about:blank",
  "title": "Bad Request",
  "status": 400,
  "detail": "size must be between 1 and 100",
  "instance": "/api/recommendations"
}
```

| Status | Meaning |
|--------|---------|
| 400 | Invalid pagination parameters |
| 401 | Missing or invalid JWT |
| 500 | Unexpected server error |
